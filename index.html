Welcome

Welcome to the maybe-annual BillGO Capture-The-Flag ("CTF") exercise. We're happy to you chose to participate and test or hone your skills as a reverse engineer. Please keep in mind that the goal of the BillGO CTF is to have fun -- if you ever feel like you're gettting stuck or frustrated (in a bad way), do not hesitate to reach out to #ctf on Slack and ask for help.

What is a CTF?

A capture-the-flag exercise is one or more problems that each involve delving into the source code or compiled code of a website, program or mobile application to retrieve a secret value such as a password, token or key -- known as the "flag".

What do I need?

There are a few tools that will make your life a lot easier as a reverse engineer.

First, be sure you are comfortable using the Chrome debugger tools (or the tools of your preferred browser). It's useful to be able to look at HTML, Javascript and CSS files, along with looking at network traffic (for the most part the Network tab in Chrome will suffice.)

Here are some other useful tools that may come in handy along the way!

  • Fiddler is a great HTTP traffic sniffer
  • JADX let's you take a peek at the code for Android apps
  • Wireshark takes it a step further, and lets you inspect all network traffic

Outside of these tools, be sure to keep a scrap pad around (whether digital or a dead tree), and don't be afraid to set breakpoints, write test scripts or program to test a hypothesis!

How deep should I dig?

All the answers to the exercises should be available by looking at client-side code. If at some point you are considering attempting to find a backdoor into the service you've gone (way) too far. Similarly, you should never need to find vulnerabilities or utilize any security issues to find a "solution".
